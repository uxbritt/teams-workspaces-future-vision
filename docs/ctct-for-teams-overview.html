Executive Summary

CTCT for Teams is the proposed name for the product currently known internally as "Indirect" — specifically Franchise/Multi-Location. The offering enables customers to access and manage multiple Direct accounts under one umbrella.

Current Gaps

1

Limited Visibility & Navigation

Users can only see teams immediately beneath them. Larger multi-tier organizations must impersonate repeatedly.

→ Fast access to any workspace in hierarchy

2

No Roll-up Reporting

Hubs must impersonate individual workspaces to determine performance. No org-level analysis possible.

→ True roll-up reporting with comparisons

3

Inconsistent Terminology

Multiple terms: multi-account, parent, partner, child, sub-account, grandchild.

→ Folder = grouping, Workspace = team

4

Irrelevant Dashboard

Current dashboard shows partner metrics irrelevant for franchise customers.

→ Franchise-specific metrics and top performers

Terminology

Folders

Navigation & access control. Groups workspaces for rollup. NOT workspaces themselves.

Workspaces

Where work happens. Direct accounts for campaigns, contacts, marketing.

Tags

Flexible reporting. Cross-cutting groupings independent of folder structure.

What Worked

  • • Pricing page clarity
  • • Hub vs. workspace terminology
  • • "Hub and spoke" mental model
  • • Navigation confidence

Pain Points

  • • Workspace/hub switch discoverability
  • • Terminology gaps
  • • Context visibility
  • • Scalability (25–100+ workspaces)

Overall: Confidence rated 4/5. Primary barriers are discoverability and scale handling — not conceptual understanding.
